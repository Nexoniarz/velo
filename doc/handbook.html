This handbook is the full language reference for Velo. Velo is built on the LuaJIT runtime; it shares Lua 5.1 semantics and the full standard library, but uses a different surface syntax. If you already know Lua, the Keyword Reference and Operators sections show everything that changed.

1. Comments

Important: Velo uses // for single-line comments and /* ... */ for block comments. The Lua -- sequence is not a comment in Velo — it means decrement. 

// This is a single-line comment

/* This is a
   multi-line block comment */

x--   // This DECREMENTS x, it is NOT a comment!

2. Variables and Types

Built-in types

Velo has the same eight types as Lua 5.1:

  • nil — the absent/unset value
  • boolean — true or false
  • number — double-precision floating-point (or integer via FFI/cdata)
  • string — immutable byte sequence, UTF-8 friendly
  • table — associative array (the universal data structure)
  • function — first-class callable value
  • userdata — opaque C pointer
  • thread — coroutine handle

Use type(v) to get the type name of any value as a string.

Scope and declaration

local x = 10          // block-scoped local
y = 20                // global (avoid in large programs)
local a, b = 1, 2    // multiple assignment
local s = "hello"    // string literal
local t = true       // boolean
local n = nil        // explicit nil

Locals are lexically scoped to the nearest enclosing block ({ }). Globals live in the global table _G.

String literals

local s1 = "double quotes"
local s2 = 'single quotes'
local s3 = [[
  long string — spans
  multiple lines
]]

// Escape sequences (same as Lua):
// \n  \t  \r  \\  \"  \'  \0  \xHH  \u{HHHH}

These operators do not exist in standard Lua. They are Velo additions.

Logical

OperatorMeaning
andLogical and (short-circuit, returns value)
orLogical or (short-circuit, returns value)
notLogical not

Note: and and or return one of their operands, not a boolean. a or b returns a if it is truthy, otherwise b. This is a common idiom for default values: local x = arg or 0.

Only false and nil are falsy; zero and the empty string are truthy.

Bitwise (via bit.*)

Velo does not have bitwise syntax operators. Use the built-in bit module: 

local bit = require("bit")
local x = bit.band(a, b)   // bitwise AND
local y = bit.bor(a, b)    // bitwise OR
local z = bit.bxor(a, b)   // bitwise XOR
local n = bit.bnot(a)       // bitwise NOT
local s = bit.lshift(a, 3)  // left shift
local r = bit.rshift(a, 3)  // logical right shift

4. Control Flow

All blocks use curly braces { }. There is no then, do, or end in Velo.

if / else if / else

if x > 0 {
    print("positive")
} else if x < 0 {
    print("negative")
} else {
    print("zero")
}

Important: else if must appear on the same line as the closing } of the previous block, as shown above.

while

local i = 0
while i < 10 {
    print(i)
    i++
}

repeat / until

local i = 0
repeat {
    print(i)
    i++
} until i >= 10

Numeric for

// for var = start, limit [, step] { ... }
for i = 1, 10 {
    print(i)
}

for i = 10, 1, -1 {
    print(i)
}

Generic for

// Iterate over a table with pairs()
local t = {x = 1, y = 2, z = 3}
for k, v in pairs(t) {
    print(k .. " = " .. v)
}

// Iterate over a sequence with ipairs()
local arr = {"a", "b", "c"}
for i, v in ipairs(arr) {
    print(i, v)
}

Note on 0-based tables: Velo table constructors are 0-based, so {"a", "b", "c"} stores "a" at index 0, "b" at index 1, "c" at index 2. However, ipairs() still starts at index 1 (Lua convention). Use a plain numeric for i = 0, #arr - 1 to iterate 0-based sequences.

break

for i = 1, 100 {
    if i > 5 {
        break
    }
    print(i)
}

goto and labels

for i = 1, 10 {
    if i == 5 {
        goto continue
    }
    print(i)
    ::continue::
}

5. Functions

Basic functions

function add(a, b) {
    return a + b
}

// Anonymous function stored in a variable
local mul = function(a, b) {
    return a * b
}

print(add(2, 3))   // 5
print(mul(4, 5))   // 20

Multiple return values

function minmax(t) {
    local lo, hi = t[0], t[0]
    for i = 1, #t - 1 {
        if t[i] < lo { lo = t[i] }
        if t[i] > hi { hi = t[i] }
    }
    return lo, hi
}

local lo, hi = minmax({3, 1, 4, 1, 5, 9, 2, 6})
print(lo, hi)   // 1   9

Varargs

function sum(...) {
    local args = {...}
    local total = 0
    for i = 0, #args - 1 {
        total += args[i]
    }
    return total
}

print(sum(1, 2, 3, 4))   // 10

Closures

function counter(start) {
    local n = start or 0
    return function() {
        n++
        return n
    }
}

local c = counter(10)
print(c())   // 11
print(c())   // 12
print(c())   // 13

6. Tables

Tables are Velo's only compound data structure. They serve as arrays, dictionaries, sets, objects, and modules.

Array-style tables (0-based)

Velo difference: Table constructors start at index 0, not index 1. {10, 20, 30} stores 10 at [0], 20 at [1], 30 at [2]. 

local fruits = {"apple", "banana", "cherry"}
print(fruits[0])   // apple
print(fruits[1])   // banana
print(#fruits)     // 3

fruits[3] = "date"
print(#fruits)     // 4

Dictionary-style tables

local person = {
    name = "Alice",
    age  = 30,
    city = "Warsaw",
}

print(person.name)        // Alice
print(person["age"])      // 30
person.email = "a@b.com"  // add new key
person.city = nil         // remove key

Mixed tables

local mixed = {
    "first",          // index 0
    "second",         // index 1
    x = 10,           // key "x"
    [100] = "sparse", // explicit numeric key
}

Metatables

Metatables let you override operators and define custom behaviors for tables. 

local Vec = {}
Vec.__index = Vec

function Vec.new(x, y) {
    return setmetatable({x=x, y=y}, Vec)
}

function Vec.__add(a, b) {
    return Vec.new(a.x + b.x, a.y + b.y)
}

function Vec:length() {
    return math.sqrt(self.x^2 + self.y^2)
}

local v1 = Vec.new(3, 4)
local v2 = Vec.new(1, 2)
local v3 = v1 + v2
print(v3.x, v3.y)       // 4  6
print(v1:length())       // 5.0

7. Object-Oriented Programming

Velo provides built-in class and new keywords for OOP. Under the hood they compile to standard metatable-based Lua objects.

Defining a class

class Animal {
    // Constructor — same name as the class
    function Animal(name, sound) {
        self.name  = name
        self.sound = sound
    }

    function speak() {
        print(self.name .. " says " .. self.sound)
    }

    function getName() {
        return self.name
    }
}

Creating instances

local dog = new Animal("Rex", "Woof")
local cat = new Animal("Mittens", "Meow")

dog.speak()    // Rex says Woof
cat.speak()    // Mittens says Meow
print(dog.getName())   // Rex

self and this

Inside class methods, self and this are equivalent — both refer to the current instance. Use whichever you prefer. 

class Counter {
    function Counter(start) {
        this.value = start or 0
    }

    function increment() {
        self.value++       // self and this are the same
    }

    function get() {
        return this.value
    }
}

Inheritance

Velo does not have a dedicated inheritance keyword. Use standard metatable chaining for inheritance: 

class Shape {
    function Shape(color) {
        self.color = color
    }

    function describe() {
        print("A " .. self.color .. " shape")
    }
}

class Circle {
    function Circle(color, radius) {
        Shape.Shape(self, color)   // call parent constructor
        self.radius = radius
    }

    function area() {
        return math.pi * self.radius ^ 2
    }
}

// Wire up inheritance
setmetatable(Circle, {__index = Shape})
setmetatable(Circle.prototype or {}, {__index = Shape.prototype or {}})

local c = new Circle("red", 5)
c.describe()                            // A red shape
print(string.format("%.2f", c.area())) // 78.54

8. Modules

Loading modules

local math   = require("math")
local string = require("string")
local json   = require("dkjson")   // third-party module

Defining a module

// mylib.velo
local M = {}

function M.greet(name) {
    return "Hello, " .. name .. "!"
}

function M.farewell(name) {
    return "Goodbye, " .. name .. "."
}

return M

// main.velo
local mylib = require("mylib")
print(mylib.greet("World"))     // Hello, World!
print(mylib.farewell("World"))  // Goodbye, World.

9. Error Handling

pcall — protected call

local ok, result = pcall(function() {
    return 10 / 0
})

if ok {
    print("Result:", result)
} else {
    print("Error:", result)
}

xpcall — with error handler

local function handler(err) {
    return debug.traceback("Error: " .. err, 2)
}

local ok, msg = xpcall(function() {
    error("something went wrong")
}, handler)

if not ok {
    print(msg)
}

error — raise an error

function divide(a, b) {
    if b == 0 {
        error("division by zero")
    }
    return a / b
}

local ok, val = pcall(divide, 10, 0)
print(ok, val)   // false   ...:division by zero

error(msg, level): level controls where in the call stack the error is attributed (default 1 = the function that called error, 2 = its caller, 0 = no location info).

10. String Patterns

Velo uses Lua's pattern matching (not full regular expressions). 

local s = "hello world 42"

// Find a pattern
local start, finish = string.find(s, "%d+")
print(start, finish)   // 13  14

// Match and capture
local word = string.match(s, "%a+")
print(word)   // hello

// All matches
for w in string.gmatch(s, "%a+") {
    print(w)    // hello, world
}

// Replace
local result = string.gsub(s, "%a+", "X")
print(result)   // X X 42

Common pattern classes:
