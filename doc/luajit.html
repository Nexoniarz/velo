Velo is a modern scripting language powered by the » LuaJIT engine. It keeps LuaJIT's world-class JIT performance and C interop while replacing the traditional Lua syntax with a cleaner, more familiar style.

What’s New in Velo

{ } blocks += -= *= /= ++ −− != not-equal class / new 0-based tables
  • Brace blocks — if x > 0 { ... } instead of then/end
  • Compound assignment — x += 1, x -= 1, x *= 2, x /= 2
  • Increment/decrement — x++ and x--
  • Not-equal operator — != instead of ~=
  • Line comments — // comment instead of -- (which is now decrement)
  • OOP builtins — class Foo { ... }, new Foo(), self/this
  • 0-based table constructors — {a, b, c} puts a at index 0

Quick Example

// greeter.velo
class Greeter {
    function Greeter(name) {
        self.name = name
        self.count = 0
    }

    function greet() {
        self.count++
        print("Hello #" .. self.count .. ", " .. self.name .. "!")
    }
}

local g = new Greeter("Velo")
for i = 1, 3 {
    g.greet()
}

$ velo greeter.velo
Hello #1, Velo!
Hello #2, Velo!
Hello #3, Velo!

Performance

Velo inherits LuaJIT’s state-of-the-art tracing JIT compiler. The Velo frontend compiles to the same bytecode, which is then JIT-compiled to native machine code at runtime. You get LuaJIT’s speed — one of the fastest dynamic language runtimes ever built — with a cleaner source syntax.

Alternatively, compile your script ahead-of-time:

  • --compile — produce a portable .veloc bytecode file
  • --compile --exec — produce a self-contained native ELF/PE executable
  • --lua — run with JIT disabled (pure fast interpreter)
